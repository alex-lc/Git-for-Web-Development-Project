    1. What is Semantic HTML? 
      Semantic HTML is HTML that has meaning that can easily be understood by us humans.

    2. What is HTML used for?
      HTML is used for displaying static content on our pages.

    3. What is an attribute and where do we put it? 
      An attribute provides additional information for an HTML element. It always comes after the tag with an equals sign: <a href="#">example</a>

    4. What is the h1 tag used for? How many times should I use it on a page?
      The h1 tag is used for the main heading of the web page. This should only be used once.

    5. Name two tags that have required attributes
      The a tag and img tag have required attributes.

    6. What do we put in the head of our HTML document?
      Information needed for the web page that doesn't need to be displayed to the user is within the head. This includes linking to
      external stylesheets, scripts, and also detailing meta data about the web page.

    7. What is an id? 
      An id in CSS is a unique id for HTML elements. An id should only be used once on a page, and is often used for navigation purposes.

    8. What elements can I add an id to?
      You can add an id to any HTML element.

    9. How many times can I use the same id on a page? 
      You should only use the same id once on a page.

    10. What is a class?
      A class is used to style HTML properties with CSS.

    11. What elements can I add a class to? 
      You can add a class to any HTML elements as well.

    12. How many times can I use the same class on a page?
      Classes, unlike id's, can be used multiple times (as many as you want) on the same page.

    13. How do I get my link to open in a new tab?
      <a href="#" target="_blank">link</a>

    14. What is the alt attribute used for?
      The alt tag is used to display a description in the absence of an image, as well as provide information for screen readers.

    15. How do I reference an id?
      You reference an id with # -> #id_example

    16. What is the difference between a section and a div
      A section is the HMTL 5, more semantic and meaningful dividing HTML element. Sections should be used to clearly organize content. 
      Divs can then be used within a section for further styling and separation.

    17. What is CSS used for? 
      CSS is used to style HTML and web pages.
  
    18. How to we select an element? Example - every h2 on the page
      To select an element, you would use its name, followed by the required brackets for CSS: h2 { }
      
    19. What is the difference between a class and an id? - Give me an example of when I might use each one
      A class is used to style HTML elements and can be used multiple times. An id however is used mainly for creating a unique marker for an HTML
      element for things like hyperlinking to a specific section of a web page. Id's can only be used once.

    20. How do we select classes in CSS?
      You select classes in CSS using . -> .example_class

    21. How do we select a p element with a single class of “human””?
      p.human { }

    22. What is a parent child selector? When would this be useful? 
      div > p for example would select all paragraphs that have a div as a parent. This is useful when you need to apply styling to
      all of the children of a parent.

    23. How do you select all links within a div with the class of sidebar?
      .sidebar a { }

    24. What is a pseudo selector?
      A pseudo selector is used on a class to style a special state of the class, like hover -> a:hover { }

    25. What do we use the change the spacing between lines?
      We use the line-height: CSS property.

    26. What do we use to change the spacing between letters?
      We usue the letter-spacing: CSS property.

    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
      We use the text-transform property -> uppercase, lowercase, and capitalize 

    28. How do I add a 1px border around my div that is dotted and black?
      div { border: 1px dotted black; }

    29. How do I select everything on the page? 
      You select everything on the page using the wildcard * { }

    30. How do I write a comment in CSS?
      You write a comment in CSS using /* COMMENT GOES HERE */

    31. How do I find out what file I am in, when I am using the command line? 
      You use the 'pwd' command.

    32. Using the command line - how do I see a list of files/folders in my current folder?
      You use the 'ls' command.

    33. How do I remove a file via the command line? Why do I have to be careful with this?
      You use the 'rm' command. You have to be careful using this command because it cannot be undone.

    34. Why should I use version control? 
      Version control is helpful because it allows you to track every single change, big or small.

    35. How often should I commit to github?
      It is recommended to commit often - 20 minutes

    36. What is the command we would use to push our repo up to github? 
      You'd use the 'git push' command.

Stretch Questions

    1. What is the difference between an inline element and a block element?
      An inline element sits on the same line as other content, while a block element will always be put on a new line.

    2. What happens when an element is positioned absolutely? 
      If an element is positioned absolutely, then it gets taken out of the content flow, making all the other elements disregard that element as
      if it didn't exist.

    3. How do I make an element take up only the amount of space it needs but also have the ability to give it a width?
      width: 100%;

    4. Name 3 elements that are diplay block by default, 2 elements that are display inline by default and 1 element that is display inline-block by default
      Block:
        * p
        * nav
        * table
      Inline:
        * span
        * em
      Inline-Block:
        * img

    5. In your own words, explain the box model. What is the fix for the box model? 
      The box model is the formatting that every single HTML element has. From the outside in, it goes margin, border, padding, and then content.
